Designing for the Web: Images

This resource will help you understand the differences between different image formats and when to use each one. See the sidebar for other Designing for the Web resources.

Image Formats

JPGs (AKA "JPEGs")

Nowadays, most of the photographs we take are digital files from the get-go. They're usually saved as JPGs on our devices, and we can easily share them to popular photo-sharing sites like Flickr, Picasa or Instagram. A JPG is a compressed bitmap format (developed by the Joint Photographic Experts Group). The JPG format is LOSSY, which means that each time a JPG is saved, it's pixels reorganize and the image compresses further. If you compress a JPG enough times, you'll end up with a single color. JPGs are best for photographs and complex images, and for the Web, 80% compression will be a smaller file size that is just as sharp as 100%.

PNGs

PNG stands for Portable Network Graphics. Unlike JPGs, PNGs are LOSSLESS, so no matter how many times you compress them, they retain their sharpness. The PNG format has gained popularity in recent years because it can replace the GIF for flat color graphics and it's robust enough for photographs and complex images, so it can be used instead of a JPG in many instances.

GIFs

GIF stands for Graphics Interchange Format. It is LOSSLESS, but only has 216 colors. On the web, it's advisable to use this format only when making animated GIFs, and use PNGs for flat color images instead.

Background Tiles

With HTML and CSS, you can repeat a small image over and over again to create a background. These images are called "background tiles". When you access online material, your browser loads everything on the page. If you use one image and tile it over and over, the browser only has to load the image once.
