Designing for the Web: Design

This page is a short guide defining the basic design principles we find in graphic and web design. This page also gives you tips on how to use these principles and elements. The "Additional Resources" section has links to modules and makes that can help you teach these principles.

Made by Laura Hilliger

Principles of Design

Page Elements

Page Elements act as visual cues to guide the user through your content. You can use various page elements to draw attention to certain content and create a hierarchy within a page. Here are some of the various page elements you might want to use:

  • Identification/Logo: communicates overarching goal (if done well)
  • Fonts/Webfonts: typeface used to be a user-defined preference - using a typeface that wasn't default meant that the user might not have it on their computer. However, today we use a wide variety of webfonts, which we style through CSS.
  • Masthead/credits: can bring a human element to an otherwise business-oriented site
  • Headlines: help create visual organization
  • Subheads/subtitles: help create visual organization
  • Pull-quotes, lead-ins and kickers: inspire deeper content dive
  • Artwork/photographs: a picture is worth a thousand words.
  • anchors/links: cross linking within the site, helpful jumping to content, keep users on the site
  • Breadcrumbs: so users know where they are

Order your content and choose elements that underline your overarching goals.

Interface design

  • Icons: can be used as visual guidance around a site. There are, arguably, several standard icons. For example, a trash can usually means delete, a folder is usually a collection of files) and so on.
  • Logic versus reality: There are times when it is wise to use standard iconography or standard layouts, and times when either no standard exists or the existing standard is undesirable. In the end, the best thing to do is listen to your users. After all, they're who you're building for.
  • Using metaphor: when designing for the web, you can think outside the box of common layouts and themes. You are free to design using metaphor or other creative perspectives because you can build anything you want.
  • Grids as Guidance: In the West, we read from left to right. In design, try not to confuse the viewer by following a grid of some sort.
  • Grid Design Methods: a fixed grid is one which stays at a certain width regardless of whether the user extends his or her browser window whereas a fluid grid extends and contracts when the user extends the window.

Typography

  • Designing for the Reader:
    • Reading with ease: shorter lines and a 1.5 line-height (also called "leading") are easier to read. Depending on the font, you might also want to increase or decrease kerning, the space between letters.
    • Webfonts: the most extensive open webfont library is at fonts.google.com. Simply choose a font, click the "Quick Use" button and copy and paste the fonts URL into the <head> tag.
    • Columns as Control: use columns to help keep large blocks of text easily readable. It's easier for a user to scroll up and down than left to right.
    • The blech factor: Basically, the idea with type is to make it easier to read, not harder.
      • CAPITALIZATION: ALL CAPS IN LARGE BODIES OF TEXT ARE HARD TO READ
      • Centering: large blocks of centered text are also difficult to read.
  • Cascading Style Sheets: Because there are multiple ways to assign CSS styles to elements, styles can "stack" on top of one another. For example, if you set a global style for a <p> tag, then assign a particular instance of that tag with a class, defined in the styles section of your page, and then place an inline style in the tag itself, the element will apply the global style, then the class style, then the inline style. This is what is meant by "cascading"
  • Graphics as text With the popularity of webfonts, there's really no reason to create images of text. Doing so makes the text less accessible – screen readers will only read the graphical text if you've included the "alt" attribute, and users will not be able to copy and paste.

Color

  • hexidecimals is a six digit number that represents colors. In CSS it begins with a # followed by a combination of numbers and letters that represent a color. The first two characters represent red, the next two green, and the final two characters represent the blues in the color. #FF0000 is red, #00FF00 is green and #0000FF is blue. #FFFFFF is white and #000000 is black.
  • Color Association – shared understandings:http://www.sensationalcolor.com/color-meaning-symbolism-and-psychology/
  • Color Terms
    • Hue: the color part of a color, such as red or blue.
    • Saturation: the intensity of that color
    • Brightness: the amount of lightness or darkness in a color
    • Neutral Colors: Black, white, or gray; also known as non-chromatic hues.
    • Chromatic Hues: not black, white or gray.
    • Monochromatic Color: Color combinations using tints and shades of a single hue.
  • Choosing Effective Colors
  • Helpful sites: Visibone Color Lab will help you see which colors look good together. It also gives you the hexadecimal number for your css! , Color Blenderallows you to set an initial color, and the blender will show you matching colors.

Consistency

Consistency is creating and following rules (and exceptions to these rules), using restraint in design and standard placement for repeating elements (such as navigation). Creating consistency leads to a better user experience.
